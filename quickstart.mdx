## Prerequisites

Before you begin, ensure you have the following installed on your system:

- **Node.js**: Version 20 or higher (required for Next.js 14)
- **Package Manager**: npm, yarn, or pnpm
- **Git**: For cloning the repository

<Info>
  Vicenta uses Node.js 20 types as specified in the `@types/node` dependency.
</Info>

## Installation

<Steps>
  <Step title="Clone the repository">
    Navigate to the Vicenta source code directory:
    
    ```bash
    cd workspace/source/front
    ```
    
    <Note>
      If you're starting from scratch, clone the Vicenta repository first.
    </Note>
  </Step>

  <Step title="Install dependencies">
    Install all required packages using your preferred package manager:
    
    <CodeGroup>
      ```bash npm
      npm install
      ```
      
      ```bash yarn
      yarn install
      ```
      
      ```bash pnpm
      pnpm install
      ```
    </CodeGroup>
    
    This will install the following core dependencies:
    
    - `next@14.2.6` - Next.js framework
    - `react@^18` - React library
    - `react-dom@^18` - React DOM rendering
    - `typescript@^5` - TypeScript compiler
    - `tailwindcss@^3.4.1` - Tailwind CSS framework
    
    <Tip>
      The installation may take a few minutes depending on your internet connection.
    </Tip>
  </Step>

  <Step title="Start the development server">
    Run the development server with hot module replacement:
    
    <CodeGroup>
      ```bash npm
      npm run dev
      ```
      
      ```bash yarn
      yarn dev
      ```
      
      ```bash pnpm
      pnpm dev
      ```
    </CodeGroup>
    
    The server will start on `http://localhost:3000` by default.
  </Step>

  <Step title="View the application">
    Open your browser and navigate to:
    
    ```
    http://localhost:3000
    ```
    
    You should see the Vicenta home page with the Next.js welcome screen.
    
    <Info>
      The page will automatically reload when you make changes to the source code.
    </Info>
  </Step>
</Steps>

## Development Workflow

### Editing the Home Page

The main application page is located at `src/app/page.tsx`. Any changes you make to this file will be reflected immediately in the browser:

```tsx src/app/page.tsx
export default function Home() {
  return (
    <main className="flex min-h-screen flex-col items-center justify-between p-24">
      {/* Your content here */}
    </main>
  );
}
```

### Modifying the Layout

The root layout in `src/app/layout.tsx` controls the overall structure and metadata:

```tsx src/app/layout.tsx
export const metadata: Metadata = {
  title: "Create Next App",
  description: "Generated by create next app",
};

export default function RootLayout({
  children,
}: Readonly<{
  children: React.ReactNode;
}>) {
  return (
    <html lang="en">
      <body className={inter.className}>{children}</body>
    </html>
  );
}
```

<Tip>
  Update the metadata in `layout.tsx` to customize your application's title and description.
</Tip>

### Global Styles

Global CSS and Tailwind directives are defined in `src/app/globals.css`. This file includes:

- Tailwind base styles
- Custom CSS variables
- Global utility classes

## Building for Production

When you're ready to deploy, create a production build:

<Steps>
  <Step title="Create the build">

      
      ```bash yarn
      yarn build
      ```
      
      ```bash pnpm
      pnpm build
      ```
    </CodeGroup>

      ```bash yarn
      yarn start
      ```

  </Step>
</Steps>

<Note>
  The production build includes optimizations like minification, tree-shaking, and automatic code splitting.
</Note>

## TypeScript Path Aliases

Vicenta is configured with path aliases for cleaner imports. Instead of relative paths, use the `@/` prefix:

```typescript
// Instead of this:
import { Component } from '../../../components/Component';

// Use this:
import { Component } from '@/components/Component';
```

The alias is configured in `tsconfig.json`:

```json tsconfig.json
{
  "compilerOptions": {
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```

## Troubleshooting

### Port Already in Use

If port 3000 is already in use, you can specify a different port:

```bash
npm run dev -- -p 3001
```

### TypeScript Errors

Run the TypeScript compiler to check for type errors:

```bash
npx tsc --noEmit
```

### Clear Next.js Cache

If you encounter build issues, try clearing the Next.js cache:

```bash
rm -rf .next
npm run dev
```
