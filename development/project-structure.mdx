## Overview

Vicenta follows the Next.js 14 App Router convention with a clean, organized structure. The application uses TypeScript for type safety and follows modern React patterns.

## Root Directory Structure

```
front/
├── public/              # Static assets
├── src/                 # Application source code
├── .eslintrc.json      # ESLint configuration
├── .gitignore          # Git ignore rules
├── next.config.mjs     # Next.js configuration
├── package.json        # Dependencies and scripts
├── postcss.config.mjs  # PostCSS configuration
├── tailwind.config.ts  # Tailwind CSS configuration
└── tsconfig.json       # TypeScript configuration
```

## Source Directory (`src/`)

The `src/` directory contains all application code:

```
src/
└── app/
    ├── layout.tsx      # Root layout component
    ├── page.tsx        # Home page component
    └── globals.css     # Global styles and Tailwind directives
```

<Info>
  The `app/` directory uses the Next.js 14 App Router, which provides file-system based routing with React Server Components by default.
</Info>

## Key Files

### App Directory

<Accordion title="layout.tsx - Root Layout">
  **Location:** `src/app/layout.tsx`
  
  The root layout wraps all pages and defines the HTML structure:
  
  ```tsx
  import type { Metadata } from "next";
  import { Inter } from "next/font/google";
  import "./globals.css";
  
  const inter = Inter({ subsets: ["latin"] });
  
  export const metadata: Metadata = {
    title: "Create Next App",
    description: "Generated by create next app",
  };
  
  export default function RootLayout({
    children,
  }: Readonly<{
    children: React.ReactNode;
  }>) {
    return (
      <html lang="en">
        <body className={inter.className}>{children}</body>
      </html>
    );
  }
  ```
  
  **Key features:**
  - Implements font optimization using `next/font/google`
  - Exports metadata for SEO
  - Applies Inter font globally via className
</Accordion>

<Accordion title="page.tsx - Home Page">
  **Location:** `src/app/page.tsx`
  
  The home page component uses Next.js Image optimization and Tailwind CSS styling:
  
  ```tsx
  import Image from "next/image";
  
  export default function Home() {
    return (
      <main className="flex min-h-screen flex-col items-center justify-between p-24">
        {/* Page content */}
      </main>
    );
  }
  ```
  
  **Key features:**
  - Server Component by default (no "use client" directive)
  - Uses `next/image` for optimized image loading
  - Responsive design with Tailwind CSS utilities
</Accordion>

<Accordion title="globals.css - Global Styles">
  **Location:** `src/app/globals.css`
  
  Global stylesheet that imports Tailwind directives and defines CSS custom properties:
  
  ```css
  @tailwind base;
  @tailwind components;
  @tailwind utilities;
  
  :root {
    --foreground-rgb: 0, 0, 0;
    --background-start-rgb: 214, 219, 220;
    --background-end-rgb: 255, 255, 255;
  }
  
  @media (prefers-color-scheme: dark) {
    :root {
      --foreground-rgb: 255, 255, 255;
      --background-start-rgb: 0, 0, 0;
      --background-end-rgb: 0, 0, 0;
    }
  }
  ```
  
  See the [Styling Guide](/development/styling) for more details.
</Accordion>

## Public Directory

The `public/` directory contains static assets that are served from the root URL:

```
public/
├── next.svg     # Next.js logo
└── vercel.svg   # Vercel logo
```

<Note>
  Files in `public/` are accessible at the root path. For example, `public/next.svg` is available at `/next.svg`.
</Note>

## Routing Convention

Next.js 14 uses file-system based routing in the `app/` directory:

| File | Purpose |
|------|--------|
| `page.tsx` | Defines a route's UI (becomes publicly accessible) |
| `layout.tsx` | Shared UI for a segment and its children |
| `loading.tsx` | Loading UI for a segment (optional) |
| `error.tsx` | Error UI for a segment (optional) |
| `not-found.tsx` | 404 UI for a segment (optional) |

### Example Route Structure

<Tip>
  To create a new route, add a folder with a `page.tsx` file:
  
  ```
  src/app/
  ├── page.tsx          → / (home)
  ├── about/
  │   └── page.tsx      → /about
  └── dashboard/
      ├── layout.tsx    → Shared layout
      └── page.tsx      → /dashboard
  ```
</Tip>

## Server vs Client Components

By default, all components in the `app/` directory are **React Server Components**:

<CardGroup cols={2}>
  <Card title="Server Components" icon="server">
    - Default in App Router
    - Run on the server
    - Can access backend resources
    - Reduce client bundle size
  </Card>
  
  <Card title="Client Components" icon="browser">
    - Require `"use client"` directive
    - Run in the browser
    - Can use hooks and interactivity
    - Access to browser APIs
  </Card>
</CardGroup>

```tsx
// Server Component (default)
export default function ServerComponent() {
  return <div>I run on the server</div>;
}

// Client Component (requires directive)
"use client";
import { useState } from "react";

export default function ClientComponent() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(count + 1)}>{count}</button>;
}
```

## Path Aliases

The project uses TypeScript path aliases for cleaner imports:

```typescript
// Instead of relative paths:
import Component from "../../../components/Component";

// Use absolute imports with @:
import Component from "@/components/Component";
```

<Info>
  The `@/*` alias is configured in `tsconfig.json` and maps to `./src/*`.
</Info>

## Build Output

When you run `npm run build`, Next.js generates:

```
.next/
├── cache/           # Build cache
├── server/          # Server-side code
├── static/          # Static assets with hashes
└── types/           # Generated TypeScript types
```

<Note>
  The `.next/` directory is excluded from version control via `.gitignore`.
</Note>

## Best Practices

<Steps>
  <Step title="Organize by Feature">
    As your app grows, consider organizing by feature:
    
    ```
    src/
    ├── app/
    ├── components/      # Reusable UI components
    ├── lib/            # Utility functions
    └── types/          # TypeScript type definitions
    ```
  </Step>
  
  <Step title="Use Co-location">
    Keep related files close to where they're used. Next.js only routes files named `page.tsx`, so you can safely co-locate other files in the `app/` directory.
  </Step>
  
  <Step title="Leverage Server Components">
    Use Server Components by default and only add `"use client"` when you need interactivity or browser APIs.
  </Step>
</Steps>
